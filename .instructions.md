# Project: Jenkins Pipeline Learning (Expense Tracker)

## What This Project Is
A **simple Expense Tracker app** (Next.js + TypeScript, localStorage only) used as a vehicle to learn enterprise CI/CD pipelines. The app is intentionally simple — the pipeline and infrastructure are the real learning.

## Tech Stack
- **App**: Next.js 14+ with TypeScript, Tailwind CSS, localStorage (NO database)
- **CI/CD**: Jenkins (runs in Docker locally)
- **Code Quality**: SonarQube Community Edition (runs in Docker locally)
- **Security Scans**: Trivy (free) + OWASP Dependency-Check (free)
- **Phase 1 Deploy**: Vercel (free hobby tier)
- **Phase 2 Deploy**: Red Hat OpenShift / Kubernetes

## Environments
- **UAT**: `develop` branch → preview/staging deployment
- **Production**: `main` branch → production deployment (requires approval gate)

## Key Conventions
- All infrastructure runs in Docker containers locally
- Jenkinsfile uses declarative pipeline syntax
- Branch strategy: `develop` → UAT, `main` → Prod
- Tests must pass before any deployment
- SonarQube Quality Gate must pass before deployment
- CRITICAL security vulnerabilities block deployment

## File Structure
- `docs/` — Step-by-step learning guides (numbered 00-09)
- `jenkins/` — Jenkins Docker setup (custom image + plugins)
- `docker-compose.yml` — Runs Jenkins + SonarQube locally
- `Jenkinsfile` — Main CI/CD pipeline
- `Jenkinsfile.vercel` — Vercel-specific deployment pipeline
- `Dockerfile` — App containerization for OpenShift
- `sonar-project.properties` — SonarQube scanner config
- `src/` — Next.js Expense Tracker app
- `__tests__/` — Test files
